Page-Level Object Methods
Page-level object methods are used in the Imposition Script module in order to manipulate the content of sheets.

Note  If you do not see the get methods during scripting in Inspire Designer, it is because the option Show getters in the Workflow Options dialog is not checked.
Tip  In Inspire Designer, these methods use meter (m) as a default measurement unit. You can use the Math Methods to convert the parameters to meters or to convert the results to required units. e.g. frommm converts a parameter from milimeters to meters and tomm converts the result from meters to milimeters.

Color and DeviceN Color Methods
Contains all methods available for the Color and DeviceNColor objects.

Colors can be created in Layout and a DeviceN color space can be assigned to a color by selecting Multiple colorants in the Color dialog. These methods can be used to manipulate such colors.

Example  Here we read properties from either Colors or DeviceNColors into the following Data output:

function CSName(Color aColor):String    // function CSNAme to identify color space
   { 
   is(aColor.ColorSpace)
      {
      ColorSpaceType.RGB:  return "RGB";
      ColorSpaceType.CMYK:  return "CMYK";      
      ColorSpaceType.HSB:  return "HSB";
      ColorSpaceType.LAB:  return "LAB";
      }
   return "";
   }
 
function FillColor(Color aColor, Bool aInOverlay)      // funtion FillColor for single colors                          
   {
   Output_0.Colors.ColorSpaceType = CSName(aColor); 
   Output_0.Colors.DeviceNCount = aColor.DeviceNCount;
   for (Int k=0; k< aColor.DeviceNCount; k++)
      {
      if (aColor.DeviceNColor[k].isValid())
         {
         Output_0.Colors.DeviceN.DeviceNDensity = aColor.DeviceNDensity[k];
         Color c = aColor.DeviceNColor[k];
         FillDeviceNColor(aColor.DeviceNColor[k],aInOverlay);
         }
      }
   Output_0.Colors.OverwriteSpotColor = aColor.OverwriteSpotColor;
   Output_0.Colors.SeparationColorSpace = aColor.SeparationColorSpace;
   Output_0.Colors.SpotColor = aColor.SpotColor;
   Output_0.Colors.InOverlay = aInOverlay;
   Output_0.Colors.commit();                           // commit the values to the Colors array                      
   }

function FillDeviceNColor(Color aColor, Bool aInOverlay)  // function for DeviceN colors which might be one of many in any single color 
   {
   Output_0.Colors.ColorSpaceType = CSName(aColor); 
   Output_0.Colors.DeviceN.Colors.DeviceNCount = aColor.DeviceNCount;
   Output_0.Colors.DeviceN.Colors.OverwriteSpotColor = aColor.OverwriteSpotColor;
   Output_0.Colors.DeviceN.Colors.SeparationColorSpace = aColor.SeparationColorSpace;
   Output_0.Colors.DeviceN.Colors.SpotColor = aColor.SpotColor;
   Output_0.Colors.DeviceN.Colors.InOverlay = aInOverlay;
   Output_0.Colors.DeviceN.commit();                    // commit the values in the DeviceN sub-array
   }           

PageContent pc = InputSheeter[0].SheetInfo[0].PageContent;  
for (Int i=0; i<pc.ObjectCount; i++)                    // repeat for the many objects on each sheet from the single input
   {
   Object o = pc.Object[i];                             
   if (o.Type == ObjectType.DrawOverlay)                // if object is an overlay
      {
      DrawOverlayObject oo = o;
      PageContent oPc = oo.PageContent;
      for (Int j=0; j<oPc.ObjectCount; j++)            // repeat to read each overlay
         {
         Object o1 = oPc.Object[j];
         if (o1.Type == ObjectType.DrawPath)
            {
            DrawPathObject pO = o1;
            FillColor(pO.FillColor, true);            // perform function FillColor for overlay objects 
            }
         }
      }
   else if (o.Type == ObjectType.DrawPath)           // perform function FillColor for non-overlay objects 
      {
      DrawPathObject pO = o;
      FillColor(pO.FillColor, false);
      }
   }

Contains all the methods for the DrawOverlay page-level objects.

Example  This script splits the contents of a single page and outputs the overlays (that are created when processing) to separate sheets, i.e. static objects (images, shapes etc) will appear on consecutive sheets if they are in different overlays.
PageContent pc = InputSheeter_0.SheetInfo[0].PageContent;
for (Int objIndex=0;objIndex<pc.ObjectCount;objIndex++)
   {
   Object obj = pc.Object[objIndex];
   if (obj.Type!=ObjectType.DrawOverlay) continue;
   DrawOverlayObject ovl = obj;
   PageContent pcO = ovl.PageContent;
   OutputSheeter_0.addSheetWithPageContent(pcO,"Main");
   }

DrawTextObject Methods
Contains all the methods for the DrawText page-level objects.

Example  The following workflow alters the appearance text from a file which is streamed into the workflow:


for(var si=0;si<InputSheeter_0.SheetCount;si++)
   {
   var pc=InputSheeter_0.SheetInfo[si].PageContent;
   for(var oi=0;oi<pc.ObjectCount;oi++)
      {
      var obj=pc.Object[oi];
      if (obj.Type==ObjectType.DrawOverlay)
         {
         var oobj=DrawOverlayObject(obj);
         pc.removeObject(oi);
         pc.insertPageContent(oi,oobj.PageContent);
         oi--;
         continue;
         }
      if (obj.Type!=ObjectType.DrawText) continue;
      var tobj=DrawTextObject(obj);
      tobj.clearClippingPaths();
      var t1=tobj.Transformation;
      var t2=-t1;
      for(var i=0;i<tobj.GlyphCount;i++)
         {
         var pos=t1.apply(tobj.GlyphPosition[i]);
         pos.Y+=pos.X*0.02*Math.sin(pos.X*200);
         tobj.GlyphPosition[i]=t2.apply(pos);
         }
      }
   OutputSheeter_0.addSheetWithPageContent(pc,"Main");
   }

The resulting output is that the input text is given a wavy look:
